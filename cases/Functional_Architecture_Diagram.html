Frontend Module Summary

  • Login — JWT + Google OAuth
  • Dashboard — Pending queue, stats, recent cycles
  • Distributors — CRUD + bulk Excel upload
  • Month Cycles — Tabs: Upload, Results, Corrections, VCS
  • Annual Plans — FY metric plans + Excel upload
  • Schemes — Standard & BSC scheme builder
  • Corrections — Corrections + Plan Revision approval
  • VCS — Recharts ComposedChart, zone-scoped
  • Admin — User management + Audit log

API Router Summary

  • auth_router — Login, Google SSO, /me
  • distributors — CRUD, bulk upload, template
  • months — Lifecycle, weightages, slabs
  • data — 12-metric actuals upload/template
  • schemes — CRUD, submit, approve, results
  • corrections — Create, review, approve/reject
  • plan revisions — Request, review, apply
  • annual_plans — Create, upload, sync
  • vcs — Zone-scoped metric time-series
  • reports — Excel disbursement export
  • users — User CRUD (admin only)

Key Data Flows

  • Auth flow: Login → JWT → all subsequent API calls carry Bearer token
  • Upload flow: Excel → openpyxl parse → upsert DistributorPlan / DistributorActual
  • Calc flow: GET /schemes/:id/results → reads plans + actuals → computes per-distributor → returns JSON
  • Correction flow: Request created → email sent → reviewer approves via UI or email link → DB updated on HOD approval
  • VCS flow: GET /vcs?fy=X → aggregates monthly cumulative sums → returns per-metric time-series with hist N-2/N-1
  • Zone scope: ZM token → role=zonal_manager → all queries filter WHERE distributor.zone = user.assigned_zone
